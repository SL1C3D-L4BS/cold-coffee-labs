# Graphify Report — placeholder

This file is regenerated by `graphify build --out graphify-out`. Until an
operator runs the install flow in `tools/ai/README.md`, this placeholder
documents the expected structure so agents can reason about it.

## Expected sections (once generated)

- **Module map** — top-level components (`engine/`, `editor/`, `gameplay/`,
  `bld/`, `tools/`, `tests/`) with node counts.
- **Hotspots** — files with highest fan-in and fan-out.
- **Ownership edges** — which service owns which subsystem
  (director ↔ ai_runtime, material_forge ↔ ai_runtime::material_eval,
  narrative ↔ gameplay/acts, etc.).
- **Recent deltas** — symbols added/removed since the previous
  `graphify build`.

## How agents consume this file

Per `.cursor/rules/graphify.mdc`, agents read this report first when
exploring the repo. The companion `graph.json` is consumed
programmatically (e.g. via `jq`) for symbol-level queries.

## Generation command

```bash
graphify build --out graphify-out --format markdown,json
```

The post-commit hook in `tools/ai/hooks/` keeps this file fresh.
