### Wave 9F — hybrid router, offline flag, feature-gated local providers
- `bld-provider::hybrid::HybridRouter` implements `ILLMProvider` over a
  priority-ordered `Vec<Arc<dyn ILLMProvider>>`. Routing rules: skip
  `Disabled` providers, skip `requires_network(kind)` providers when
  the shared `OfflineFlag` is ON, skip providers whose `health ==
  Unhealthy && rolling_failures >= max_failures` (default 3). First
  eligible wins; on error, falls through to the next. Tracks
  `last_used()` for metrics. Streaming dispatches first-match with the
  same eligibility rules but does not mid-stream re-route.
- `OfflineFlag` is an `AtomicBool` wrapper — flip it from any thread,
  every router pickup sees the change. Slash command `/offline` is
  wired to mutate the same flag via `SessionContext::offline`; the
  router reads the flag directly.
- `bld-provider::local` lands `MistralrsProvider` and `LlamaProvider`
  stubs. Both advertise `Unhealthy` until `with_model(path)` is called
  (no weights → no pretending to work); calls surface a descriptive
  "no model configured" or "real inference path not wired in this
  build" error. The feature flags `local-mistralrs` and `local-llama`
  still gate the heavy dependencies (pinned in workspace `Cargo.toml`);
  the stub surface is always compiled so the hybrid router can be

### Non-obvious calls
- **Bridge dispatch stays synchronous.** Every tool the bridge routes
  must run on the editor main thread, so there's no value in an async
  surface here; `spawn_blocking` on the caller side handles the one
  place we cross the boundary.
- **CompositeDispatcher `route_exact` is just `route` with prefix==id.**
  Matching still goes through `starts_with`, so both use-cases share
  one code path. Exact routes placed before prefix routes give us
  priority.
- **Router failure accounting is transient.** Rolling failure counts
  reset on first success; the router never persists them across
  restarts. Claude's SDK already does exponential backoff; this layer
  only prevents cascading calls to a known-bad endpoint within a turn.
- **Local provider stubs refuse work without a model path.** A zero-
  config build that still advertises `Healthy` would quietly hide
  misconfiguration; advertising `Unhealthy` + erroring on call makes
  the router correctly skip to the next provider.
- **Replay `--bit-identical` implies `--strict` in spirit but is
  stricter** (fails on skipped events too) because audit fixtures used
  for CI are meant to be fully re-runnable; a skip there is a red
  flag, not a checkpoint.
