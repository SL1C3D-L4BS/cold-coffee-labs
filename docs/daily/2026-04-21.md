## Addendum · Phase 8 fullstack, end of day

After the Phase-7 close-out, pulled Phase 8 end-to-end in a single push
(weeks 043–047). Scope, waves, landings, and sign-off below.

### Waves shipped

**Wave 1 — `.gwscene` codec + component migration (week 043)**

- Deleted the orphaned `engine/core/scene_format.{hpp,cpp}` stub (`GWSC`
  magic, two-way conflict with ADR-0006 `GWS1`). Never wired, never
  correct.
- New module `engine/scene/` with
  - `scene_file.{hpp,cpp}` — `save_scene` / `load_scene` composing
    `ecs::save_world` + atomic temp-file-then-rename on both Win32
    (`MoveFileExW(REPLACE_EXISTING|WRITE_THROUGH)`) and POSIX
    (`rename(2)`).
  - `migration.{hpp,cpp}` — `MigrationRegistry` keyed on
    `(stable_hash, from_size)` with registered `MigrateFn` callbacks.
- `engine/ecs/serialize.{hpp,cpp}`: `LoadOptions` gains a
  `MigrateComponentFn`; `parse_payload` now defers the
  `ComponentSizeMismatch` check and `apply_payload` consults the
  callback before failing.
- `editor/bld_api/editor_bld_api.cpp` registers migrations once at
  `gw_editor_load_scene` entry; `gw_editor_{save,load}_scene` route
  through the new module. `WorldMatrixComponent` added to the live
  type registry.
- Tests: `tests/unit/scene/scene_file_test.cpp` — round-trip, atomic
  rename, CRC tamper detection, end-to-end migration from a
  hand-crafted v1 payload. All green.
- **Doctrine:** ADR-0008 landed.

**Wave 2 — float64 transforms (week 044)**

- `TransformComponent::position` widened from `glm::vec3` (12 B) to
  `glm::dvec3` (24 B → struct 56 B with 8-byte alignment padding) per
  CLAUDE.md #17.
- New `WorldMatrixComponent { glm::dmat4 world; u8 dirty; u8[7] pad; }`
  — trivially-copyable, cache-aligned, serializable.
- New `editor/scene/transform_system.{hpp,cpp}` —
  `update_transforms(World&)` walks `HierarchyComponent +
  TransformComponent`, composes world matrices in `dmat4`, writes
  `WorldMatrixComponent`. O(n) two-pass (child-index + recursive
  compose from roots); thread-local scratch buffers keep steady-state
  allocation-free.
- `editor/reflect/widget_drawer.cpp`: `kTypeDVec3` +
  `ImGui::DragScalarN(ImGuiDataType_Double, …)` so the Inspector edits
  float64 positions without silently truncating.
- `editor/panels/viewport_panel.cpp`: debug-draw calls cast
  `glm::dvec3 → glm::vec3` at the float32 debug-draw boundary (the
  renderer itself is still float32 for camera-relative draws).
- `editor/app/editor_app.cpp`: demo entities seeded with `glm::dvec3`
  positions.
- Tests: `tests/unit/scene/transform_system_test.cpp` — float64
  precision, parent/child composition, and the v1→v2 `TransformComponent`
  migration via a hand-crafted scene blob.

**Wave 3 — vscript IR + parser + interpreter (week 045)**

- `engine/vscript/ir.{hpp,cpp}` — `Type`, `Value` (variant), `Pin`,
  `Node`, `Graph`, insertion-stable vectors, `validate()` with
  disconnected-pin / unknown-edge / type-mismatch / cycle (iterative
  DFS) checks.
- `engine/vscript/parser.{hpp,cpp}` — hand-rolled lexer + recursive
  descent; text ↔ `Graph` round-trip stable to byte. Fixed two lexer
  bugs along the way: `advance()` bypassing the pending-token stash,
  and `lex_number()` greedily consuming "3." when "3.lhs" was intended.
- `engine/vscript/interpreter.{hpp,cpp}` — bottom-up demand walk,
  memoised by (node_id, pin_name). Thread-local cache, allocation-free
  on steady state.
- Tests: `tests/unit/vscript/vscript_ir_test.cpp` — 8 cases (validate
  DAG / disconnected pin / cycle; interpreter adder + select;
  round-trip; error line numbers; type-mismatch validation).

**Wave 4 — bytecode + VM + oracle match (week 046)**

- `engine/vscript/bytecode.{hpp,cpp}` — `OpCode`, `Instr` (24 B
  record), `Program` with constant pools for strings + vec3 literals +
  IOPort tables. `compile(Graph&) → Program` emits one sub-program per
  `set_output` by recursive `emit_input`.
- `engine/vscript/vm.{hpp,cpp}` — stack VM, one `switch` dispatch loop,
  `vector<Value>` operand stack reserved to `code.size()`. On `halt`
  fills unwritten outputs with declared defaults to mirror interpreter
  behaviour.
- Tests: `tests/unit/vscript/vscript_vm_test.cpp` — 5 cases, every one
  runs both the interpreter and the VM and checks output maps are
  identical. Covers adder, select, float mul, compile-time validation
  rejection, and default-fill on graphs with no `set_output`.
- **Doctrine:** ADR-0009 landed (covers the whole IR → VM → panel
  pipeline).

**Wave 5 — ImNodes projection + layout sidecar (week 047)**

- `editor/vscript/vscript_panel.{hpp,cpp}` — the `VScriptPanel`
  `IPanel`. Owns its own `ImNodesContext` so multiple panels don't
  stomp globals.
- Pin-id packing: `(node_id << 16) | (pin_index << 1) | is_output` so
  every pin is a unique ImNodes int id.
- Layout sidecar: `<path>.layout.json` written as plain
  `{"<node_id>": [x, y], …}` via a ~40-line hand-rolled reader/writer.
  `nlohmann::json` deliberately kept out of the editor TU for this
  trivially tiny format.
- Panel layout: IR text buffer on the left (ground-truth, editable),
  ImNodes canvas on the right. "Parse" re-builds the graph from text;
  "Run" evaluates through the reference interpreter and renders the
  output map in an ImGui table.
- Docked in the centre area alongside Viewport.
- **Editing via nodes is explicitly out of scope for Phase 8** —
  Phase 9 will add `CommandStack`-backed add-node / wire-edge /
  rename-port operations.

### Non-obvious calls

- Chose **callback-in-LoadOptions** over a pre-pass rewriter for
  component migration. Writing the migrator at the ECS layer keeps the
  archetype walk intact; the pre-pass alternative duplicated parse
  logic and would have to evolve in lockstep with ADR-0006 §2.2.
- Kept the VScript panel **view-only** — editing through nodes is
  Phase 9. Phase 8 only needs to prove IR round-trip + ImNodes
  projection. Shipping a half-editable graph would invite
  "why-doesn't-this-work" papercuts that the `CommandStack` groundwork
  will cleanly resolve.
- Shared sub-expression memoisation in the VM compiler — **not** in
  Phase 8. The stack-machine bookkeeping isn't worth it for 50-node
  graphs, and Phase 9's JIT will get SSA memoisation for free.
